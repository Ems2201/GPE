Como Funciona o GPE?

Hoje iremos ver como funciona o GPE, e como utilizarmos ele para realizar a exibição das vistas frontais de alguns conjuntos de manobra

O gerador de painéis elétricos, ou GPE, é um gerador de vistas frontais de conjuntos de manobra que ajuda os usuários a tentarem alocar as unidades funcionais de um conjunto de manobra com o menor número possível de colunas. Ele possui um pequeno banco de dados com unidades funcionais retiradas do catálogo do ABB (MNS-MCC), para tornar as simulações mais realistas, pois os valores de corrente, potência e altura podem variar de fabricante para fabricante.

Além disso o GPE é dividido em duas partes, o formulário de dados, isto é, onde deve ser realizado um passo a passo para inserir as unidades funcionais, e a tabela de dados onde é exibido cada unidade funcional que foi adicionada. Porém, por ser um software open-source, isto é, um software de código aberto, é permitido colocar outros fabricantes, com outros dados, apenas modificando o arquivo “data.js”, sendo este, o arquivo com os dados.

Como simular um Conjunto de Manobra no GPE?

A primeira coisa a ser feita para gerar uma vista frontal é selecionar o Conjunto de Manobra, no caso de exemplo, CCM.

Logo após, deve ser selecionado qual unidade deseja inserir, no caso do exemplo vamos inserir um disjuntor de entrada de um CCM de 2000 A, mas poderia ser qualquer unidade funcional. O CCM é um centro de controle de motores, composto principalmente de partidas de motores. Logo, devemos escolher o Disjuntor CCM da lista de unidades funcionais, informar sua corrente no campo corrente e informar a quantidade no campo quantidade.

Feito isso devemos indicar qual barramento está unidade será inserida, no caso de CCMs, o padrão é utilizar apenas um barramento, no caso Barramento A. Haveria um barramento B se houvesse duas entradas separadas por um tie breaker.

Agora é necessário clicar no botão de adicionar UF, que logo após de ser clicado irá aparecer na tabela de entradas a unidade funcional adicionada.

Pronto, já será possível cadastrar o painel e ser encaminhado para a vista frontal.

Após entender como realizar as simulações é possível também, mudar as ordenações das unidades funcionais conforme os algoritmos do menu das vistas frontais.

Lembrete: para cada algoritmo, há uma resposta diferente, onde os seguintes algoritmos realizam os seguintes procedimentos:

  • Best-Fit: Seleciona a unidade funcional correspondente e a aloca na coluna com menor espaço possível.
  • Best-Fit Menor Para o Maior:Igual ao Best-Fit, mas ordena as unidades funcionais de maneira crescente na entrada.
  • Best-Fit Maior Para o Menor:Igual ao Best-Fit, mas ordena as unidades funcionais de maneira decrescente na entrada.
  • Best-Fit Melhorado: Faz o mesmo procedimento do maior para o menor, porém ele realiza um cálculo do número mínimo de colunas a partir da soma das alturas, e caso o número mínimo de colunas não seja atingido, ele reordena as unidades funcionais das colunas que não estão cheias. Além disso, ele tem um botão que permite ao usuário mudar de posição as unidades funcionais.
  • Bin-completion: Outra derivação do Best-Fit, porém ele deve primeiro preencher uma coluna para depois iniciar uma coluna com outra unidade funcional, ou seja, ele somente insere uma unidade funcional assim que ele preencher totalmente a coluna, e caso não consiga preencher totalmente a coluna, passa para próxima coluna e próxima unidade funcional correspondentes
  • Next-Fit: Ele coloca cada unidade funcional na coluna atual se houver espaço suficiente; caso contrário, abre uma nova coluna. Logo, mesmo que haja colunas anteriores à atual que contenham espaço, ele só pode alocar as unidades funcionais.

Entendido isso, é necessário compreender que certos algoritmos terão exibições diferentes ou até mesmo iguais a depender da maneira que foi inserido os dados, além de que, fazendo que alguns algoritmos exibam mais colunas e outros menos colunas.
